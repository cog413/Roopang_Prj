## 🔴 Phase 1 — 기본 완성도 (우선순위 높음)

> 현재 구현된 것들의 "빠진 부분"을 메우는 단계. 사용자가 실제로 쓸 수 있는 최소 완성 상태.

| 항목 | 세부 내용 | 비고 |
|---|---|---|
| **존버 Index 실시간 갱신** | setInterval로 1분마다 KPI 재계산 | 현재 로드 시 1회만 계산됨 |
| **게임 시간당 1판 제한** | SOP 제2조 이행. 게임별 마지막 플레이 시각 체크 | DB 또는 localStorage 기반 |
| **케어 Index 100% 달성 조건** | 현재 1종(상호작용)만 있어 최대 50%. 2종 이상 활동 타입 추가 | 먹이주기, 꾸미기 등 |
| **점수/포인트 로직 정비** | PRD 기준 reward = baseReward × difficulty × performance 구현 | 현재 단순 카운트 방식 |
| **게임 포인트 정규화** | 수도쿠·2048 각각 0~1 performance 스케일 통일 | 게임 간 보상 공정성 |

## 🟡 Phase 2 — 사람이 모이는 기능 (핵심 재방문 유도)

> 랭킹과 캐릭터 — 이게 들어가야 DAU가 의미 있어짐.

| 항목 | 세부 내용 | 비고 |
|---|---|---|
| **랭킹 시스템** | 일/주/월 개인 랭킹("이달의 사원"), 회사 랭킹("매출 순위") | ReadMe 시트에 랭킹 카드 추가 |
| **랭킹 워딩 적용** | 전체 랭킹 UI에 직장인 컨셉 워딩 일관 적용 | context 정책 참고 |
| **캐릭터 3종 완성** | 현재 A(정장)/B(캐주얼) → C(막내) 추가 + 각 캐릭터 CSS 애니메이션 정성 구현 | 감정별 반응 애니 포함 |
| **펫 레벨링 시스템** | 포인트 누적 → 경험치 → 레벨업("승진") → 외형 변화 | PRD 4.1 기반 |
| **펫 스타일링 시스템** | 아이템 장착 (넥타이, 안경, 의상 등) — 포인트 소모 | 코스메틱 경제 기반 |
| **친구 추천 씬 구조 추가** | 온보딩 또는 로그인 페이지에 "누구의 추천?" 이메일 입력 필드 | 로직 없어도 구조 먼저 |
| **`?ref=` 파라미터 보존** | OAuth 리다이렉트 시 ref 파라미터 state에 담아 유지 | 추천 귀속 기반 인프라 |

## 🟡 Phase 3 — 완성도 + 수익화 기반

> 실제로 쓸 만하고 돈이 되는 구조.

| 항목 | 세부 내용 | 비고 |
|---|---|---|
| **모바일 웹 최적화** | 768px 반응형, 2048 스와이프 제스처, 수도쿠 온스크린 넘버패드 | 터치 최소 44×44px |
| **가짜 업무표 강화** | 각 게임 탭의 사이드 대시보드를 더 자연스러운 실제 업무 자료처럼 정교화 | 직장인 몰입도↑ |
| **Google AdSense 준비** | `Data_Visual_Widget` 자리에 AdSense 코드 삽입 구조 완성 | 심사 타이밍에 맞춰 |
| **결제 시스템 기초** | 코스메틱 아이템 카탈로그 + purchases 테이블 + 결제 수단 연동 (Toss/Stripe) | 펫 스킨 판매 첫 타겟 |
| **추천 리워드 로직** | ref 귀속 확인 → 첫 게임 완료 후 referrer/referree 양측 포인트 지급 | Phase 2 구조 선행 필요 |

## 스콥 차이 메모 (AI 추정 vs 실제 계획)

| 항목 | AI 기존 추정 | 실제 계획 |
|---|---|---|
| 캐릭터 종류 | 2종 (A/B) | 3종 + 정성스러운 감정별 애니메이션 |
| 케어 Index | "2종 활동 추가" 수준 | 본격 펫 레벨링·스타일링 시스템으로 확장 |
| 랭킹 | 단순 점수 리스트 | 직장인 워딩 + 일/주/월 갱신 정책 포함 |
| 수익화 | 코스메틱 아이템 막연히 언급 | Google AdSense + 직접 결제 병행, 구체적 분기 |
| 글로벌 | 언급 없음 | 재활용성 원칙부터 설계에 내재화 |
| 마케팅 | 언급 없음 | 친구 추천 포인트 리워드 + 온보딩 씬 구조화 |
| 모바일 | 언급 없음 | 터치 제스처, 반응형 레이아웃, 게임 UX 전면 대응 |
